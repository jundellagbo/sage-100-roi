# Sage 100 ROI — WooCommerce Connector

A WordPress plugin that connects and synchronizes your ecommerce platform between **Sage 100** and **WordPress WooCommerce**.

## Features

- **OAuth 2.0 authentication** with the Sage 100 / ROI Consulting API (Microsoft Azure AD client-credentials flow)
- **Encrypted credential storage** — API keys and tokens are stored using AES-256 encryption via `FSD_Data_Encryption`
- **Two-environment support** — toggle between *development* and *production* OAuth credentials from the admin panel
- **Data synchronization** — products, product images, customers, orders, and in-process items
- **Draft product categories** — hide categories from the shop without deleting them
- **Force login** — redirect unauthenticated visitors to the login page automatically
- **Custom WooCommerce account dashboard** — lands users on their Orders page and adds an *Ordering Portal* menu link
- **WP-CLI support** — trigger syncs from the command line
- **Auto-updates** via GitHub releases using the [Plugin Update Checker](https://github.com/YahnisElsts/plugin-update-checker)

## Requirements

- WordPress 5.0+
- WooCommerce 5.0+
- PHP 7.4+

## Installation

1. Download or clone this repository into your `wp-content/plugins/` directory.
2. Run `composer install` inside the plugin directory to install PHP dependencies.
3. Activate the plugin from the **Plugins** screen in WordPress admin.
4. Navigate to **Tools → Sage 100 ROI** to enter your API credentials.

## Configuration

All settings are managed from **Tools → Sage 100 ROI** in the WordPress admin.

| Setting | Description |
|---|---|
| OAuth Token URL | The Microsoft OAuth 2.0 token endpoint (e.g. `https://login.microsoftonline.com/{tenant}/oauth2/v2.0/token`) |
| Client ID / Secret / Scope | Development OAuth credentials |
| Client ID / Secret / Scope (Production) | Production OAuth credentials |
| Use Production | Toggle between dev and production credentials |
| Stop sync flags | Individually pause synchronization of items, images, customers, or orders |
| Reset page counters | Restart a sync from page 1 for each data type |

After saving, the plugin will verify connectivity to the Sage 100 API and display a success or error notice.

## REST API

The plugin registers custom WooCommerce REST API endpoints. Requests must include two headers for authentication:

| Header | Description |
|---|---|
| `Sage-ROI-X-App-ID` | The application ID generated by the plugin |
| `Sage-ROI-X-App-Secret` | The application secret generated by the plugin |

The App ID and App Secret are regenerated each time credentials are saved.

## WP-CLI

After activating the plugin you can trigger sync operations from the command line:

```bash
wp sage-roi <command>
```

Run `wp help sage-roi` for a list of available sub-commands.
